## 2026-06-29

### 推送 WinLauncher 子集到 GitHub
- 用户要求将 `WinLauncher/`, `WinLauncher.sln`, `assets/` 三个路径上传到 `https://github.com/LEISHIQIANG/WinLauncher.git`，其他路径全部忽略
- 远程仓库原本只有 LICENSE，等同首次 push
- 实施流程：
  1. `cp -r .git` 备份到 `%TEMP%/ql_wl_filter_git`（防 filter-branch 出错）
  2. `git stash push -u` 暂存工作区未提交改动
  3. `git filter-branch -f --index-filter 'git rm -r --cached --ignore-unmatch .gitignore .workbuddy docs scripts' --prune-empty --tag-name-filter cat -- --all` 重写历史
  4. `git remote add origin https://github.com/LEISHIQIANG/WinLauncher.git`
  5. `git push -u origin main --force`
- 验证：远程仓库根目录只有 `WinLauncher.sln`, `WinLauncher/`, `assets/` 三项，共 128 个文件
- 关键决策：用户指定"只上传三个"指 git 树内容（commit 历史中只保留这三个路径），工作区的 .vs/, x64/, test_resource.obj, build_log.txt 等不被追踪的中间产物保持原样不动

### 经验教训
- `git filter-branch` 在有未暂存改动时会拒绝运行，必须先 `git stash` 或 commit
- 路径包含空格的 Windows 项目用 `cd 'C:/...'` 单引号包裹最稳
- `git-filter-repo` 未安装也没关系，filter-branch 足够用；如要静默警告加 `FILTER_BRANCH_SQUELCH_WARNING=1`
